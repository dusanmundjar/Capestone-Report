Data acquisition and data summary

Data are obtained from link: “http://d396qusza40orc.cloudfront.net/dsscapstone/dataset/Coursera-SwiftKey.zip”. File is unziped and only en-US data is analysed. We have three files that we saved in folder corpus under project’s main directory. Names of the files are:

  • en_US.blogs.txt

  • en_US.news.txt

  • en_US.twitter.txt

Cleaning the data

First, we have to read necessary packages to generate text mining corpus.

library(NLP)
library(tm)
library(SnowballC)
library(RWeka)

Then we generate corpus to do the analysis of the text data.

data<-VectorSource(data)
corpus<-VCorpus(data)

To be sure that our corpus is clean we can remove everythig that different from letters followed by space.

removeNumPunct <- function(x) gsub("[^[:alpha:][:space:]]*", "", x)
corpus <- tm_map(corpus, content_transformer(removeNumPunct))

We coud do analysis more pricise by removing punctuation, numbers, set tellers to lowers, remove white spaces that are not useful. Effect is similar, but maybe in some case difference may be significant. In our case, in our opinion, it is not.

We do the steeming of the words, too.

Usually, stopwords are also removed, but our goal is to predict next word so we need to keep all words.

corpus <- tm_map(corpus, removePunctuation)
corpus <- tm_map(corpus, removeNumbers)
corpus <- tm_map(corpus, content_transformer(tolower))
corpus <- tm_map(corpus, stripWhitespace)
corpus <- tm_map(corpus, stemDocument)

Exploratory analysis

For exploratory analysis we use RWeka, package that connects R and Weka. Weka has a large variety of algorithms included and we use those for text mining - extracting n-grams from text.

Preparing data for Weka.

corpus<-data.frame(text=unlist(sapply(corpus, `[`, "content")), stringsAsFactors=F)

Tokenisation

Genrating tokens with 1, 2, 3 and 4 words.

token.1w <- NGramTokenizer(corpus, Weka_control(min = 1, max = 1))
token.2w <- NGramTokenizer(corpus, Weka_control(min = 2, max = 2, delimiters = " \\r\\n\\t.,;:\"()?!"))
token.3w <- NGramTokenizer(corpus, Weka_control(min = 3, max = 3, delimiters = " \\r\\n\\t.,;:\"()?!"))
token.4w <- NGramTokenizer(corpus, Weka_control(min = 4, max = 4, delimiters = " \\r\\n\\t.,;:\"()?!"))

Modeling ideas

Goal of the projest is to build a simple model for prediction of the word that writter wants to write. For that purpuse we have intention to:

  • use n-grams to if text is potentialy part of 2-gram, 3gram, 4-gram,
  • find unfrequent words, and try to find high correlated unfriequent words.
  • all together is formedad as Markov Chain where function returns three word with highest probability of occurance.
